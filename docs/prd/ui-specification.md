## Overview

This document specifies the exact UI layout, components, interactions, and workflows for the Streamlit-based Surebet Accounting System interface.

---

## Design Principles

1. **Single-User Focus**: No authentication, no role-based access control
2. **Local-First**: All data operations are instant (no network latency)
3. **Clarity Over Aesthetics**: Functional, data-dense interface for rapid decision-making
4. **Audit Trail Visibility**: Every money-impacting action is traceable
5. **No Silent Actions**: All critical operations require explicit confirmation

### Components

#### Upload Manual Bet Panel

**Screenshot File Picker**:
- Accepted formats: PNG, JPG, JPEG, PDF
- Max size: 10 MB
- Shows preview thumbnail after selection

**Associate Dropdown**:
- Lists all associates by `display_alias` (alphabetical)
- Searchable

**Bookmaker Dropdown**:
- Filtered by selected associate's bookmakers
- If no associate selected, disabled

**Note Field**:
- Optional free text (max 500 chars)
- Placeholder: "e.g., WhatsApp bet from morning session"

**Import & OCR Button**:
- Disabled until screenshot + associate + bookmaker selected
- On click:
  - Show loading spinner ("Processing OCR...")
  - Create bet with `status="incoming"`, `ingestion_source="manual_upload"`
  - Run OCR pipeline
  - Add to queue below
  - Clear form

---

#### Incoming Bets Queue

**Summary Counters**:
- **Waiting review**: Count of `status="incoming"`
- **Approved today**: Count of `status="verified"` created today
- **Rejected today**: Count of `status="rejected"` created today

**Bet Cards**:

Each bet displays as an expandable card.

**Header**:
- Confidence badge: âœ… (normalization_confidence â‰¥ 0.8) or âš  (< 0.8)
- Associate alias â€¢ Bookmaker name â€¢ Ingestion source (Telegram / Manual Upload)
- Timestamp received (UTC)

**Body**:
- Screenshot preview (clickable to enlarge)
- Event dropdown (searchable, shows canonical events sorted by kickoff time)
- Market details (editable for low-confidence)
  - Market code (dropdown from `canonical_markets`)
  - Period scope (dropdown: Full Match / First Half / Second Half)
  - Line value (text input, validated as decimal)
  - Side (dropdown: OVER / UNDER / YES / NO / TEAM_A / TEAM_B)
- Financial details (editable for low-confidence)
  - Stake (text input, validated as decimal)
  - Odds (text input, validated as decimal)
  - Payout (auto-calculated: `stake * odds`)
  - Currency (read-only from associate's `home_currency`)

**Footer**:
- **Approve** button: Green, primary
  - On click:
    - Validate all fields
    - Set `status="verified"`
    - Log changes to `verification_audit`
    - Attempt matching into surebet
    - Remove from queue (slide out animation)
    - Show success toast: "Bet approved âœ“"
- **Reject** button: Red, secondary
  - On click:
    - Show confirmation modal: "Reject this bet? This cannot be undone."
    - Set `status="rejected"`
    - Remove from queue
    - Show success toast: "Bet rejected"

**Editable Fields** (for low-confidence bets):
- All fields are inline-editable
- Changes auto-save to bet record (not ledger yet)
- Visual indicator when field is edited (yellow highlight)

## Common UI Patterns

### Color Scheme

| Element | Color | Hex |
|---------|-------|-----|
| Success / Good | Green | `#28a745` |
| Warning / Attention | Orange | `#ffc107` |
| Error / Unsafe | Red | `#dc3545` |
| Primary Action | Blue | `#007bff` |
| Secondary Action | Gray | `#6c757d` |
| Background | White | `#ffffff` |
| Card Border | Light Gray | `#dee2e6` |

### Typography

- **Font Family**: System default (Streamlit default)
- **Headers**: Bold, 18-24px
- **Body**: Regular, 14-16px
- **Code/Numbers**: Monospace for stakes, odds, amounts

### Buttons

**Primary Button** (Approve, Confirm, Generate):
- Background: Blue (#007bff)
- Text: White
- Hover: Darker blue

**Secondary Button** (Cancel, Reject):
- Background: Gray (#6c757d)
- Text: White
- Hover: Darker gray

**Danger Button** (Reject, Delete):
- Background: Red (#dc3545)
- Text: White
- Hover: Darker red

### Badges

- âœ… High Confidence: Green background, white text
- âš  Low Confidence: Orange background, white text
- âŒ Unsafe: Red background, white text
- ğŸŸ¡ Marginal: Yellow background, dark text

### Modals

- Centered overlay
- Semi-transparent backdrop (40% black)
- White card with shadow
- Title, body, footer with buttons
- Close on backdrop click (except for critical confirms)

### Toast Notifications

- Top-right corner
- Auto-dismiss after 3 seconds
- Success: Green border
- Error: Red border
- Info: Blue border

### Loading States

- Spinner for async operations (OCR, settlement)
- Disable buttons during processing
- Show "Processing..." text

### Keyboard Navigation

- All interactive elements accessible via Tab
- Enter to submit forms
- Escape to close modals

- WCAG AA compliance for text/background contrast
- Minimum 4.5:1 for body text
